### 5.4 — Model Fallback Router

> Concept: Requests fail over from a primary model provider to a secondary provider under 5xx, 429, or timeout conditions, with circuit breakers to stop flapping.
> Attack surface: Partial outages, schema drift between providers, and shared-budget exhaustion can turn fallback into cascading failure.
> Eval target: Zero user-visible errors when primary is down

---

## Section 1 — Core Intuition

Provider availability is not binary; it degrades by region, model, or quota bucket, and naïve fallback chains simply bounce traffic between unhealthy backends until both are overloaded. The mechanism here is provider abstraction with per-provider circuit breakers and a shared load budget; the analogy is power grid failover where you isolate the bad line before switching sources, because without that isolation the backup fails under the same surge.

ATTACK: Cascading provider overload
VECTOR: Primary starts timing out, router immediately doubles load on the secondary, and both quotas collapse.
EXPLOIT PAYLOAD: 100 requests/sec during a regional primary outage.
CONSEQUENCE: Total outage instead of partial degradation.
MITIGATION:

The router tries the primary provider first. On any `APIError`, `RateLimitError`, or `APITimeoutError`, it falls back to Anthropic. If both fail, `AllProvidersUnavailableError` is raised:
```python
def complete(prompt: str) -> CompletionResult:
    try:
        text = call_openai(prompt)
        return CompletionResult(provider="openai", content=text, fallback_used=False)
    except (APIError, RateLimitError, APITimeoutError):
        log.warning("openai_unavailable_falling_back")
    try:
        text = call_anthropic(prompt)
        return CompletionResult(provider="anthropic", content=text, fallback_used=True)
    except (APIConnectionError, APIStatusError, AnthropicRateLimitError):
        raise AllProvidersUnavailableError("all providers exhausted")
```

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Teams implemented fallback with nested try/except and no breaker, so the primary kept flapping into the request path.
FIX: `@circuit(failure_threshold=3, recovery_timeout=60)`
• FAILURE: Both providers were treated as infinite capacity and exhausted together during a regional incident.
FIX: `if self._shared_budget <= 0: raise AllProvidersUnavailableError("shared provider budget exhausted")`
• FAILURE: Fallback events were invisible, so operators saw only rising latency without knowing requests were being rerouted.
FIX: `log.warning("fallback_event", provider="anthropic", error_type=type(primary_error).__name__, fallback_latency_overhead_ms=120)`
• FAILURE: Breaker state was held in memory only, making every process restart forget the outage history.
FIX: `self._state_store.save(ProviderState(...))`
• FAILURE: Prompt size checks happened after the router had already attempted both providers.
FIX: `enforce_token_budget(6000, prompt)`
