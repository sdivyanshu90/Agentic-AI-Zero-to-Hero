### 5.3 — Fleet Scaling

> Concept: Horizontally scaled agent pods coordinate through Redis to stay below provider rate limits before 429s happen.
> Attack surface: Distributed bursts, forged client identities, and Redis pool exhaustion can bypass backpressure or crash the limiter.
> Eval target: Zero 429 errors under 50 req/s load

---

## Section 1 — Core Intuition

Single-process concurrency control stops mattering the moment you have multiple pods, because every replica thinks it is being polite while the provider sees an aggregate flood. The mechanism is a shared rate limiter with preemptive jitter and explicit load shedding; the analogy is a freeway metering system that spaces cars before the merge, because without it every pod accelerates into the same bottleneck and the provider becomes the traffic cop.

ATTACK: Redis connection pool exhaustion
VECTOR: High concurrency opens too many Redis sockets and the limiter fails open.
EXPLOIT PAYLOAD: 50 pods each opening unbounded connections during a traffic burst.
CONSEQUENCE: The rate limiter disappears exactly when needed most.
MITIGATION:

Concurrency is tracked in a thread-safe counter. Requests are rejected before they reach the model when the in-flight count is at capacity:
```python
with limiter_lock:
    if state.in_flight >= state.max_concurrency:
        raise RuntimeError("load shed: concurrency limit reached")
    state.in_flight += 1
try:
    result = run_model_call(prompt)
finally:
    with limiter_lock:
        state.in_flight -= 1
```

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Teams used a per-process semaphore and assumed it represented cluster-wide load.
FIX: `pipeline.incr(key, 1)`
• FAILURE: The limiter failed open when Redis connection churn exhausted the client pool.
FIX: `return redis.Redis.from_url(os.getenv("REDIS_URL"), decode_responses=True, max_connections=100)`
• FAILURE: Pods continued sending traffic at 99% quota because the limiter only shed after the hard limit.
FIX: `if int(window_count) > int(0.85 * RATE_LIMIT): ... time.sleep(delayed_ms / 1000)`
• FAILURE: Engineers accepted user-supplied identity headers into the limiter key and created an easy bypass.
FIX: `state = acquire_slot(request_id)`
• FAILURE: 429s were treated as transient model failures and retried into a larger outage.
FIX: `if state.status == "shed": raise OverCapacityError("shared rate limit exhausted")`
