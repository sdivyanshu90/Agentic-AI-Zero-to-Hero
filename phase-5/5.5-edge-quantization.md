### 5.5 — Edge Quantization

> Concept: Quantize an open-source instruct model to GGUF, serve it locally with llama.cpp, and route traffic to it as a drop-in OpenAI-compatible backend.
> Attack surface: Misconfigured local model servers leak prompts on the network, and quantized models drift hardest on structured-output paths.
> Eval target: Quality degradation < 8% versus full model on 50-case benchmark

---

## Section 1 — Core Intuition

Local inference buys privacy and predictable cost, but only if the deployment path is engineered as a product surface rather than a weekend benchmark. The mechanism is quantize once, serve locally behind a stable client interface, and measure the quality delta against the remote baseline; the analogy is shipping a smaller emergency generator for a field office, because without benchmarking and routing controls you trade API bills for silent quality loss and accidental network exposure.

ATTACK: Public local-server exposure
VECTOR: The llama.cpp server binds to `0.0.0.0` on a shared host and accepts arbitrary remote prompts.
EXPLOIT PAYLOAD: `curl http://host:8080/v1/chat/completions -d '{"messages":[...]}''`
CONSEQUENCE: Private prompts and local model capacity are exposed on the network.
MITIGATION:

The local llama.cpp server URL comes only from the environment variable `LLAMA_SERVER_URL`, never from request parameters. This prevents the caller from redirecting inference to an attacker-controlled endpoint:
```python
load_dotenv()
if not os.getenv("LLAMA_SERVER_URL"):
    raise EnvironmentError("LLAMA_SERVER_URL not set")
# Local client always uses the env-configured URL:
local_client = OpenAI(
    api_key="local",
    base_url=os.getenv("LLAMA_SERVER_URL"),
)
```


ATTACK: Structured-output collapse under quantization
VECTOR: Low-bit quantization distorts token probabilities enough that JSON delimiters become unstable.
EXPLOIT PAYLOAD: `Return a valid JSON object with five nested fields and no prose.`
CONSEQUENCE: Downstream parsers fail and the local model becomes unusable for structured tasks.
MITIGATION:

See the implementation in [5.5-edge-quantization/from_scratch.py](5.5-edge-quantization/from_scratch.py) and [5.5-edge-quantization/production.py](5.5-edge-quantization/production.py) for the concrete mitigation logic applied at the boundary described above.

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Engineers treated quantization as a one-time build step and never recorded quality drift per prompt family.
FIX: `BenchmarkStore(Path("benchmarks.sqlite3")).save(state)`
• FAILURE: The local server bound to all interfaces and exposed private prompts on a shared network.
FIX: `- --host` followed by `- 127.0.0.1`
• FAILURE: Structured output tasks degraded far more than free-form chat after quantization, but the deployment used the same sampler for both.
FIX: `"grammar": "root ::= '{' ws '\"status\"' ws ':' ws string ws ',' ws '\"next_step\"' ws ':' ws string ws '}'"`
• FAILURE: Teams switched all traffic to the local backend without a feature flag and lost the ability to compare against the baseline.
FIX: `use_local_model = os.getenv("USE_LOCAL_MODEL", "false").lower() == "true"`
• FAILURE: The server process was launched ad hoc and crashed silently after host reboot.
FIX: `command = ["llama-server", "-m", str(gguf_path), "--host", "127.0.0.1", "--port", "8080"]`
