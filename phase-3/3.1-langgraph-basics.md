### 3.1 — LangGraph Basics

> Concept: Model an agent workflow as explicit typed state plus deterministic graph edges instead of hidden control flow.
> Attack surface: Unbounded loops, missing state fields, and tool-output injection become harder to debug when graph state is implicit.
> Eval target: Graph transition correctness > 0.95 on 20 workflow cases

---

## Section 1 — Core Intuition

Most “agents” are just nested `if` statements plus retries, which works until the workflow needs pause, resume, checkpointing, or a second branch that shares state. The mechanism of LangGraph is simple and useful: make state explicit, make nodes explicit, and make edges explicit; the analogy is a circuit diagram replacing a bag of loose wires, because once the workflow is drawn as state transitions you can reason about why it stalled, looped, or skipped a step.

ATTACK: Infinite routing loop
VECTOR: A route function keeps sending state back to retrieval without a hard retry ceiling.
EXPLOIT PAYLOAD: Empty answer on every pass.
CONSEQUENCE: The graph spins forever and burns tokens.
MITIGATION:

Retry count is an explicit field in `AgentState` and is checked before each node transition. The `route_after_answer` function enforces the ceiling:
```python
def route_after_answer(state: AgentState) -> str:
    if state.retry_count >= MAX_RETRIES:
        return "terminal_failed"
    if state.answer:
        return "terminal_success"
    return "retriever"  # retry
```

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Teams used a graph library but still hid key routing logic inside helper functions no one could inspect.
FIX: `graph.add_conditional_edges("answer", route_after_answer, {END: END, "retrieve": "retrieve"})`
• FAILURE: Retry counters lived outside graph state and reset on resume, creating accidental infinite loops.
FIX: `retry_count: int`
• FAILURE: Empty inputs propagated all the way to the model call and failed expensively.
FIX: `if not question.strip(): return ToolError(...)`
• FAILURE: Operators assumed checkpointing meant correctness, but the stored state schema was incomplete.
FIX: `class AgentState(TypedDict): ...`
• FAILURE: Success criteria focused on answer text and ignored whether the workflow actually took the intended path.
FIX: `assert mean(scores) > 0.95`
