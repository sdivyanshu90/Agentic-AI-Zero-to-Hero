### 3.3 — Human-in-the-Loop Escalation

> Concept: Pause high-risk agent actions and require human approval, edit, or rejection before the workflow continues.
> Attack surface: Approval bypasses, stale approvals, and queue spoofing can turn a safety checkpoint into a false sense of control.
> Eval target: Escalation precision > 0.90 on risky-action test cases

---

## Section 1 — Core Intuition

Some actions are too expensive or too irreversible to let the model execute alone: refunds, credential changes, production deletes, or legal replies. The mechanism is a deliberate interrupt in the workflow that surfaces the proposed action and waits for a human decision; the analogy is a bank transfer above a threshold that requires a second approver, because without that pause the agent’s “confidence” becomes a substitute for accountability.

ATTACK: Approval bypass
VECTOR: High-risk actions route directly to execution when state defaults to `approve` or `low`.
EXPLOIT PAYLOAD: refund plus account deletion submitted as normal support action.
CONSEQUENCE: Irreversible actions happen without review.
MITIGATION:

Each approval record includes a SHA-256 hash of `thread_id + proposed_action`. The reviewer compares the displayed action to the stored hash before approving, so a tampered action cannot be substituted after display:
```python
def action_hash(thread_id: str, proposed_action: str) -> str:
    return hashlib.sha256(
        f"{thread_id}:{proposed_action}".encode("utf-8")
    ).hexdigest()
record = ApprovalRecord(
    thread_id=thread_id,
    action_hash=action_hash(thread_id, proposed_action),
    proposed_action=proposed_action,
    risk_level=risk_level,
)
```

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Teams called something “human in the loop” but allowed execution to proceed while review was still pending.
FIX: `interrupt_before=["approval"]`
• FAILURE: Approval applied to thread ID only and ignored whether the action text had changed.
FIX: `action_hash: str`
• FAILURE: Review queues filled with low-risk tasks and humans learned to approve everything on reflex.
FIX: `risk_level: Literal["low", "medium", "high"]`
• FAILURE: Rejections were not terminal, and the workflow retried into execution anyway.
FIX: `if record.decision == "reject": return "action rejected"`
• FAILURE: Teams measured queue throughput instead of escalation correctness.
FIX: `assert mean(scores) > 0.90`
