### 1.2 — CLI OS Assistant

> Concept: A native tool-calling agent that runs shell commands and controlled file operations inside a workspace-safe boundary.
> Attack surface: Path traversal and command misuse can turn a helpful shell agent into an arbitrary host access primitive.
> Eval target: Task completion rate > 0.90 on 50 real CLI tasks

---

## Section 1 — Core Intuition

Once the model is allowed to act, the problem stops being “can it reason” and becomes “can it act without destroying the machine.” The mechanism is explicit tool schemas, a bounded agent loop, and workspace-constrained execution for shell, reads, and writes; the analogy is giving a junior operator a badge that opens only certain doors, because without those boundaries the first prompt-injected file path turns into a host breach.

14. `safe_write_file(path, content)` calls `resolve_workspace_path` first, then writes. In the production version this function is never called directly — it is held in `pending_write` state and only executed after human approval.

ATTACK: Path traversal
VECTOR: File tool arguments use `../../` segments to escape the workspace root.
EXPLOIT PAYLOAD: `../../etc/passwd`
CONSEQUENCE: The assistant reads or writes host files outside the intended sandbox.
MITIGATION: `resolve_workspace_path` resolves the input to an absolute path and rejects anything that does not start with `WORKSPACE_ROOT`. The check runs before any filesystem operation:
```python
def resolve_workspace_path(path_str: str) -> Path:
    resolved = (WORKSPACE_ROOT / path_str).resolve()
    if not str(resolved).startswith(str(WORKSPACE_ROOT)):
        raise PermissionError(f"path {resolved} is outside workspace")
    return resolved
```

ATTACK: Shell injection through string execution
VECTOR: The agent concatenates a command string and executes through a shell.
EXPLOIT PAYLOAD: `ls; cat ~/.ssh/id_rsa`
CONSEQUENCE: The model gains arbitrary shell chaining beyond the intended tool action.
MITIGATION: `bash_exec` uses `subprocess.run` with `shell=False` and a list of arguments, so `;`, `&&`, and `|` are treated as literal arguments rather than shell operators:
```python
result = subprocess.run(
    [command, *args],
    shell=False,       # no shell interpolation
    capture_output=True,
    timeout=10,
    cwd=WORKSPACE_ROOT,
)
```

ATTACK: Unreviewed writes
VECTOR: The model decides to overwrite a file without operator approval.
EXPLOIT PAYLOAD: `write_file` call targeting a tracked source file.
CONSEQUENCE: Silent repo mutation and data loss.
MITIGATION: The production LangGraph sets `interrupt_before=["human_review"]`. The workflow pauses before the write node and only resumes when an operator explicitly calls `graph.invoke(None, config)` after reviewing the pending action:
```python
graph = builder.compile(
    checkpointer=SqliteSaver.from_conn_string("cli_agent.sqlite3"),
    interrupt_before=["human_review"],
)
```

## Section 6 — Pitfalls & Pro-Tips

• FAILURE: Teams trusted relative paths and discovered too late that `../../` resolves perfectly on Linux.
FIX: `if not str(resolved_path).startswith(str(WORKSPACE_ROOT)):`
• FAILURE: The command executor used `shell=True`, so harmless-looking tasks became shell-injection primitives.
FIX: `subprocess.run(argv, ..., shell=False, check=False)`
• FAILURE: Tool errors were raised directly and terminated the loop before the agent could recover.
FIX: `return ToolError(tool_name="bash_exec", error_type="TimeoutExpired", message=str(exc))`
• FAILURE: Write operations happened automatically during planning, and the repo drifted under prompt injection.
FIX: `interrupt_before=["human_review"]`
• FAILURE: The loop never stopped when the model kept asking for tools, so the assistant burned tokens without making progress.
FIX: `for iteration_count in range(10):`
